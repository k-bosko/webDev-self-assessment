HTML + CSS + JavaScript Self Assessment

1. Most useful tags in HTML:

  1. section - to organize main content
  2. nav - to create navigation bar
  3. div - to organize stuff
  4. a - to make a link
  5. img - to add images
  6. button - to add buttons
  7. h1-h6 - to add section headings
  8. p - to make paragraphs
  9. ul, ol, li - to organize into ordered/unordered list
  10. form, input - to accept data from users

2. Most important header tags for SEO

  • title - sets up the title that will show up in search engine result page (SERP)
  • meta name="description" - creates a short preview what the site is about in SERP
  • meta name="keywords" - provide more information about page's content for search engine. However, there's a controversy about this tag and search engines pay less attention to it nowadays.
  • meta name="robots" - sets rules for crawling and indexing pages

In addition, following tags help with SEO:

  • headings H1-H6 - structures the text for search crawler bots, help them understand what the page is about
  • img alt - helps search crawler bots to index images
  • HTML 5 semantic tags = article, section, aside, header, footer, nav - further improve communication with search crawler bots

Description based on "8 of the Most Important HTML Tags for SEO" (March 27, 2020).

6. Positioning

6.1. Relative Positioning

Relative positioning fixes an element relative to position the element would have taken in the normal HTML flow

I'm relatively positioned element

static positioning example

6.2. Absolute Positioning

Absolute positioning fixes an element in place relative to its nearest positioned ancestor (the initial containing block if there isn't one). An absolutely positioned element no longer exists in the normal document flow. Instead, it sits on its own layer separate from everything else. Usage: popup information boxes, control menus, rollover panels, UI features that can be dragged and dropped anywhere on the page and don't interfere with the layout of other elements on the page.

I'm absolutely positioned element relative to the <section> element, not the <html> element!

6.3. Fixed Positioning

Fixed positioning usually fixes an element in place relative to the visible portion of the viewport. Usage: navigation menus that are always visible

Fixed positioning example

6.4. Sticky Positioning

Sticky positioning is a hybrid between relative and fixed position. It allows a positioned element to act like it's relatively positioned until it's scrolled to a certain threshold (e.g., 10px from the top of the viewport), after which it becomes fixed. Usage: cause a navigation bar to scroll with the page until a certain point and then stick to the top of the page

Sticky positioning example
